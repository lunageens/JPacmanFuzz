Jpacman Fuzzing report - All maps

This page provides an overview of all map and string files, sorted per exit message. Certain combinations of inputs (a random map file and an action sequence) will get send in to the JPacman application, and the exit code describes how the program handled them. An exit code 0 is received if the application had a normal termination. An exit code 1 stands for a crash of the program. An exit code 10 is received when the application could recognize an invalid input and rejected it. If the JPacman program returns any other response code, this is registered as exit code -1. Note that the fuzzing program itself also categorizes certain inputs as -1 to avoid crashes. This is always indicated with a corresponding output message. For example, an action sequence and map that lets the player move to a wall cell will always let the program crash and thus return the exit code 1. Therefore, the fuzzing program checks (for certain customActionSequenceNr configurations) whether the inputs would result in invalid moves before sending the inputs to the JPacman application. If this is the case, the input will get exit code -1 and output message "Invalid move". Note that the fuzzing application will not send those inputs to the JPacman application. This is to speed up the fuzzing process.
